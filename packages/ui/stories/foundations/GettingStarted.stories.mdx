# Getting Started with @repo/ui

Welcome to the @repo/ui component library! This guide will help you get started with our design system and understand how to use our components effectively.

## Installation

```bash
# Install the UI package
pnpm add @repo/ui

# Install peer dependencies
pnpm add react react-dom
```

## Basic Usage

### Importing Components

```jsx
// Import individual components
import { Button, Input, Text } from '@repo/ui';

// Or import everything
import * as UI from '@repo/ui';
```

### Your First Component

```jsx
import { Button, Text, Box } from '@repo/ui';

function MyComponent() {
  return (
    <Box p="md" display="flex" flexDirection="column" gap="sm">
      <Text size="lg" weight="semibold">Welcome to our UI Library</Text>
      <Button tone="primary" onClick={() => alert('Hello!')}>
        Get Started
      </Button>
    </Box>
  );
}
```

## Design Principles

### 1. Accessibility First

All components are built with accessibility as a primary concern:

- **Semantic HTML**: Use appropriate HTML elements
- **Keyboard Navigation**: Full keyboard support
- **Screen Readers**: Proper ARIA labels and descriptions
- **Color Contrast**: WCAG 2.1 AA compliant color combinations
- **Focus Management**: Visible focus indicators

### 2. Design Token Driven

All visual properties use design tokens:

```jsx
// ✅ Good - uses design tokens
<Button tone="primary" size="md">Primary Button</Button>

// ❌ Avoid - hard-coded values
<button style={{ backgroundColor: '#3b82f6', padding: '16px' }}>
  Custom Button
</button>
```

### 3. Composable

Components are designed to work together:

```jsx
<Box display="flex" flexDirection="column" gap="md">
  <Text as="h2" size="lg" weight="semibold">
    Form Title
  </Text>
  <Input placeholder="Enter your email" />
  <Button tone="primary" size="lg">Submit</Button>
</Box>
```

## Best Practices

### 1. Use Semantic Elements

```jsx
// ✅ Good - semantic HTML
<Text as="h1">Page Title</Text>
<Text as="p">Paragraph content</Text>

// ✅ Also good - explicit element
<Box as="section">Section content</Box>
<Box as="article">Article content</Box>
```

### 2. Provide Accessible Labels

```jsx
// ✅ Good - accessible icons
<Icon label="Search icon">
  <path d="M15.5 14h-.79l-.28-.27C15.41 12.59 16 11.11 16 9.5 16 5.91 13.09 3 9.5 3S3 5.91 3 9.5 5.91 16 9.5 16c1.61 0 3.09-.59 4.23-1.57l.27.28v.79l5 4.99L20.49 19l-4.99-5zm-6 0C7.01 14 5 11.99 5 9.5S7.01 5 9.5 5 14 7.01 14 9.5 11.99 14 9.5 14z" />
</Icon>

// ✅ Good - decorative icons
<Icon decorative>
  <path d="M12 2l3.09 6.26L22 9.27l-5 4.87 1.18 6.88L12 17.77l-6.18 3.25L7 14.14 2 9.27l6.91-1.01L12 2z" />
</Icon>
```

### 3. Handle Loading and Error States

```jsx
function SubmitButton({ isLoading, error, onClick }) {
  return (
    <Box display="flex" flexDirection="column" gap="sm">
      <Button 
        tone="primary" 
        loading={isLoading}
        disabled={isLoading}
        onClick={onClick}
      >
        {isLoading ? 'Submitting...' : 'Submit'}
      </Button>
      
      {error && (
        <Text size="sm" tone="error">
          {error}
        </Text>
      )}
    </Box>
  );
}
```

### 4. Use Consistent Spacing

```jsx
// ✅ Good - consistent spacing
<Box display="flex" flexDirection="column" gap="md">
  <Text>Item 1</Text>
  <Text>Item 2</Text>
  <Text>Item 3</Text>
</Box>

// ❌ Avoid - inconsistent spacing
<div>
  <p style={{ marginBottom: '8px' }}>Item 1</p>
  <p style={{ marginBottom: '16px' }}>Item 2</p>
  <p style={{ marginBottom: '24px' }}>Item 3</p>
</div>
```

## Testing

### Storybook Testing

All components include comprehensive Storybook stories for:

- **Visual Testing**: Chromatic visual regression testing
- **Accessibility Testing**: axe-core integration
- **Interaction Testing**: User interaction flows
- **Responsive Testing**: Multiple viewport testing

### Unit Testing

Components are tested with Vitest and Testing Library:

```jsx
import { render, screen } from '@testing-library/react';
import { Button } from '@repo/ui';

test('Button renders with correct text', () => {
  render(<Button>Click me</Button>);
  expect(screen.getByRole('button')).toHaveTextContent('Click me');
});

test('Button handles click events', () => {
  const handleClick = jest.fn();
  render(<Button onClick={handleClick}>Click me</Button>);
  
  screen.getByRole('button').click();
  expect(handleClick).toHaveBeenCalledTimes(1);
});
```
